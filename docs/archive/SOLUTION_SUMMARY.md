## 🔧 技术方案

### 核心算法：增强贪心调度策略

#### 1️⃣ 多轮调度机制
```
每个调度时间点：
  while (货车剩余容量 > 0) and (存在可调度站点对):
    - 识别所有有余量的站点（供给）
    - 识别所有有缺口的站点（需求）
    - 选择优先级最高的配对进行调度
    - 更新货车容量和库存
```

#### 2️⃣ 未来需求预测（2-3小时展望）
```
缺口计算公式：
  deficit_i = max(0, R_i - S_{i,t}) + λ·Σ_{k=1}^{2} max(0, R_i - S_{i,t+k})
  
其中：
  R_i = 站点i的安全库存目标（容量的50%）
  S_{i,t} = 站点i在时刻t的当前库存
  λ = 0.5（未来权重系数）
```

#### 3️⃣ 距离加权优先级
```
优先级分数计算：
  priority_{ij} = (deficit_j × surplus_i) / distance_{ij}
  
调度策略：贪心选择最高分数的站点对
```

## 💡 创新点

### 1. 多轮调度机制
- **解决问题**：一次性调度可能浪费容量，实时匹配可能不及时
- **创新做法**：每小时内循环调度，充分利用货车容量
- **效果**：平均容量利用率较高

### 2. 未来需求预测
- **解决问题**：仅考虑当前缺口导致调度滞后
- **创新做法**：融入未来2-3小时的预测缺口，提前调度
- **效果**：减少后期缺货和溢出

### 3. 距离加权优先级
- **解决问题**：简单的需求权重忽视了地理成本
- **创新做法**：`priority = (demand × supply) / distance`
- **效果**：平衡需求紧迫性和运输成本

## 📚 相关公式汇总

### 费用函数
```
总费用 = 惩罚成本 + 运输成本

惩罚成本 = Σ_t [α·满桩小时数_t + β·空桩小时数_t]

运输成本 = Σ_trip [3.5×距离_trip + 2.0×车辆数_trip]
```

### 缺口与余量
```
余量 = max(0, 库存 - 安全目标)
缺口 = max(0, 安全目标 - 库存) + 未来权重×预测缺口

安全目标 = 站点容量 × 50%
```

### 优先级评分
```
priority = (目标站缺口 × 源站余量) / 两站距离

选择规则：贪心选择最高优先级配对
```
