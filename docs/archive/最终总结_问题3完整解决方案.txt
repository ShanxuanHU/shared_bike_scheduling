【增强贪心算法详解】
========================================================================================

算法三大创新点的实现：

【创新1: 多轮调度】
   程序位置: problem3_simulation.py, greedy_schedule_enhanced() 方法
   
   伪代码:
   while cap_remaining > 0:
       for each (供给站i, 需求站j):
           score[i,j] = deficit[j] × surplus[i] / distance[i,j]
       
       (best_i, best_j) = argmax(score)
       amount = min(surplus[best_i], deficit[best_j], cap_remaining)
       
       schedule(best_i → best_j, amount)
       cap_remaining -= amount
   
   关键: 外层while循环重复直到货车容量用完或无可行配对

【创新2: 未来需求预测（2-3小时）】
   程序位置: problem3_simulation.py, calculate_surplus_deficit() 方法
   
   公式:
   deficit_i = max(0, R_i - S_i,t) + λ·Σ(k=1~2)max(0, R_i - predict_S_i,t+k)
   
   参数:
   • R_i = 0.5 × C_i（安全库存 = 容量的50%）
   • S_i,t = 当前库存
   • predict_S_i,t+k = 预测的未来库存
   • λ = 0.5（未来权重）
   
   效果: 提前识别潜在的库存缺口，主动补充

【创新3: 距离加权优先级】
   程序位置: problem3_simulation.py, greedy_schedule_enhanced() 方法
   
   优先级公式:
   priority_ij = (deficit_j × surplus_i) / distance_ij
   
   含义:
   • 分子: deficit_j × surplus_i (缺货严重 × 货源充足)
   • 分母: distance_ij (距离越近优先级越高)
   
   效果: 优先调度距离近且能有效缓解缺货的方案

【融合实现】
   三项创新在greedy_schedule_enhanced()方法中完美融合：
   
   1. 计算enhanced_deficit (创新2)
   2. 识别供给站和需求站
   3. 循环(创新1)中按优先级(创新3)贪心选择最优配对
   4. 直到容量或配对对象用完
