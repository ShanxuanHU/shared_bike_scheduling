# 共享单车调度优化问题

## 题目说明

本项目围绕共享单车站点运行数据，完成了以下 4 个问题：

### 问题1：描述性分析

基于 `附件2_每小时借还记录.csv` 与 `附件1_站点基础信息.csv`，我们分析了 30 个站点在一周内的借还规律，识别出：

1. 全天典型借出峰、归还峰及持续时间；
2. 借出量最高 / 最低站点及其空间分布特征；
3. 工作日与休息日借还模式的主要差异。

### 问题2：未来 1 天需求预测

在认识借还基本规律的基础上，我们建立了对目标日期 24 小时逐站点借出量、归还量的预测模型，并在测试日上进行了精度评估。

当前实现采用：

- 小时周期特征：`hour_sin`、`hour_cos`
- 日期类型特征：`is_weekday`
- 可提前获得的历史特征：前一日同小时借还量 `lag_24`
- 训练集均值基线：同站点同小时历史均值
- 空间特征：站点聚类分区 `zone`

最终预测值采用“基线 + 残差模型”的方式得到。“基线”的引入使我们充分运用了前一题识别出的基本借还规律，避免了对借还量的直接学习，保证了较高的预测精度。

### 问题3：调度优化

基于问题2的预测结果，我们对工作日进行了“无调度 / 有调度”仿真对比，输出：

1. 总费用；
2. 满桩 / 空桩发生次数；
3. 用户无法借车 / 还车次数；
4. 用户需求满足率；
5. 调度明细与库存变化。

基于假设，当前题3实现采用离散时延调度：

- 调度在整点发车；
- 调度时间按 `t_ij = round(d_ij / v)` 取整；
- 源站发车当小时立即扣减库存；
- 目标站在 `t + t_ij` 时刻入库。

### 问题4：模型推广与灵敏度分析

在问题3的基础上，我们完成了两部分扩展：

1. 从实用性出发，新增了 5 个站点，构造扩展后的 35 站网络，并重新运行调度模型；
2. 对运输成本、搬运成本、满桩惩罚系数、空桩惩罚系数做敏感性分析。

由此，我们验证了模型的可推广性，并确认了该模型对哪些参数较为敏感。

## 典型问题

在本项目的推进过程中，我们实际遇到了不少问题。在此记录一些令我印象较深的问题及其解决办法，也许会对未来的某些同类型项目有参考价值：

### 问题1：评估失准

本项目中，我们原本采用mae, rmse，mape这3种方式评估预测模型的精度。运行后得到的mae, rmse分别在2-3和3-4之间也在预料之内，是可接受的。但这里的mape由于实际数据（作为分母）中有多个值为0，导致数值过于巨大，参考价值不高，后续删除。

### 问题2：数据分布不一致

第二题中，我们用于模型学习的数据工作日占比过高，而题目却要求我们预测星期天的借还情况。不可避免地，预测结果工作日化。虽然我们可以通过提升休息日数据的权重来平衡休息日预测的失真，但是训练数据中只有 1 天为休息日导致这种修正的实际意义也非常有限。

以后类似项目中，最好保持数据较少的一方也有足够分析的数据量。

### 问题3：启发式算法的局限

第三题建立的数学规划模型明显属于混合整数非线性优化问题（MINLP）：

- 整数性：调度车辆数、是否调度
- 非线性：惩罚函数为分段函数
- 动态性：每小时状态均有变化，且前一时刻影响后一时刻

此类问题很难精确求解，毕竟数据复杂度太高。于是我们根据类似问题的经验，选定了贪心算法。然而，贪心算法只能保证局部最优，难以达成全局最优。

因此我在想，该问题本质上还是一个带容量约束的网络流问题，是否可以尝试使用图论解决。不过，该系统具有动态性。目前我只有一个利用站点与时间设点，利用调度与否设边的初步想法，暂时没有更加具体切实的实现思路。
